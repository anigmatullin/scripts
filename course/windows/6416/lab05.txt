===========Задание 1. Создание политик изоляции ==================================

Выполните задание на dc-01

1. Создайте политику IPSec, используя предыдущую версию консоли Ipsec.

 - Откройте Start->Administrative Tools->Local Security Policy
 - Откройте раздел "IP Security Policies on Local Computer"
 - Создайте новую политику IPSec c параметрами по умолчанию.

2. Создайте политику IPSec, используя Новые средства.

 - Откройте Start->Administrative Tools->Windows Firewall with Advanced Security.
Альтернативно, можно открыт раздел "Windows Firewall with Advanced Security" в консоли "Local Security Policy".
 - Перейдите в раздел "Connection Security Rules".
 - Создайте правило с именем "Domain Isolation Policy", используя следующие параметры

	• Type: Isolation
	• Requirements: Require authentication for inbound connections and request authentication for outbound ...
	• Authentication method: Computer and user (Kerberos V5)
	• Profile: Выберите только профиль domain
	• Name: Domain Isolation Policy


===========Задание 2. Создание политик Qos ==================================

Выполните задание на dc-01

1. Создайте политику QoS

 - Запустите Group Policy Management Console (Start->Run->введите gpmc.msc)
 - Откройте редактор для ОГП "Default Domain Policy"
 - Откройте раздел Computer Configuration->Windows Settins->Policy based QoS
 - Создайте новую политику с параметрами:
	• Policy name: Backup
	• DSCP Value: 10
	• This Policy applies to: "Only applications with this executable name"; введите Backup.exe



===========Задание 3. Использование адресов IPv6==================================

Выполните задание на dc-01, cl-01

1. Получите информацию на об адресах IPv6 на dc-01
	
 - На dc-01 запустите ipconfig. Просмотрите IPv6 адрес для интерфейса "Local Area Conection".

2. Получите информацию на об адресах IPv6 на cl-01
	
 - На cl-01 запустите команду:
>netsh advfirewall firewall add rule name="allow icmp"  protocol=icmpv6 dir=in action=allow

- Запустите ipconfig. Скопируйте IPv6 адрес для интерфейса "Local Area Conection" в буфер обмена.

3. Пинг с использованием IPv6

 - На dc-01 запустите команду
>ping <IPv6 адрес dc-01>
